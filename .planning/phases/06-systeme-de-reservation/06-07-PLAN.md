What was built:
- Direct booking 3-step wizard (calendar -> address -> payment confirmation)
- Quote request flow (describe need -> provider responds -> client accepts)
- Provider booking dashboard with status tabs and action buttons
- Client "Mes reservations" page with status tabs
- Cancellation with tiered refund policy display
- Status timeline on booking detail pages
- Navigation integration in Navbar, BottomNav, and ProviderSidebar

How to verify:

**Setup:** Ensure dev server is running (`npm run dev`). Have at least one provider account with KYC approved and one active service (both FIXED and SUR_DEVIS pricing types).

**Flow 1 — Direct Booking (FIXED service):**
1. Log in as CLIENT
2. Navigate to a FIXED-price service detail page
3. Click "Reserver" — should open 3-step wizard at /services/[id]/book
4. Step 1: Select an available date on calendar (blocked dates should be grayed out), select a time slot
5. Step 2: Enter address and city
6. Step 3: Select a payment method (4 options visible + "Mode demonstration" banner), click confirm
7. Verify redirect to booking detail page showing PENDING status
8. Verify booking appears in "Mes reservations" -> "A venir" tab

**Flow 2 — Provider Accepts Booking:**
9. Log in as PROVIDER
10. Go to /provider/bookings -> "Nouvelles demandes" tab
11. Verify the new booking appears with Accept/Reject buttons
12. Click "Accepter" — status should change to ACCEPTED
13. Verify in client view: booking now shows ACCEPTED status

**Flow 3 — Quote Request (SUR_DEVIS service):**
14. Log in as CLIENT
15. Navigate to a SUR_DEVIS service detail page
16. Click "Demander un devis" — should open form at /services/[id]/quote
17. Fill in description (50+ chars), address, city, optional budget
18. Submit — should show success toast and redirect to bookings
19. Log in as PROVIDER, check "Nouvelles demandes" — quote should appear with respond form
20. Enter proposed price, click "Repondre"
21. Log in as CLIENT, check booking — quote should show RESPONDED with accept/decline
22. Accept the quote — booking should be created

**Flow 4 — Cancellation:**
23. Find a PENDING or ACCEPTED booking as CLIENT
24. Click cancel — dialog should show refund tier (100%, 50%, or 0% based on time)
25. Confirm cancellation
26. Verify booking status changed to CANCELLED

**Flow 5 — Status Progression:**
27. As PROVIDER: accept a booking, start service, complete service
28. Verify status timeline updates at each step on both client and provider detail pages

Resume signal: Type "approved" if all flows work correctly, or describe any issues found.
