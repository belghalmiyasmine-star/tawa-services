**BookingActions (`src/features/booking/components/BookingActions.tsx`):**

"use client" component. Props: `bookingId: string`, `status: BookingStatus`, `onStatusChange: () => void`.

Renders action buttons based on current status:

- **PENDING:**
  - "Accepter" (green Button) — calls acceptBookingAction(bookingId), shows success toast
  - "Rejeter" (red outline Button) — opens AlertDialog with optional reason textarea, calls rejectBookingAction({ bookingId, reason })

- **ACCEPTED:**
  - "Demarrer le service" (primary Button) — calls startBookingAction(bookingId), shows success toast
  - "Annuler" (red outline) — calls cancelBookingProviderAction from Plan 02 (import from cancel-booking.ts)

- **IN_PROGRESS:**
  - "Marquer comme termine" (green Button) — calls completeBookingAction(bookingId), shows success toast

- **COMPLETED / REJECTED / CANCELLED:**
  - No action buttons (read-only)

All buttons have loading state (useTransition or useState pending). After successful action, call onStatusChange() to trigger page refresh/revalidation.

Use useRouter().refresh() for revalidation after mutation. Show toast with appropriate success message using i18n keys.
