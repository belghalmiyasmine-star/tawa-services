**Availability API (`src/app/api/provider/availability/route.ts`):**

Create a public GET endpoint:
- Query params: `providerId` (required), `month` (YYYY-MM format, required)
- Fetch provider's Availability records (7 day slots with isActive, startTime, endTime)
- Fetch BlockedDate records for the given month
- Fetch existing bookings for provider in that month (status PENDING or ACCEPTED) to show occupied slots
- Return JSON:
```json
{
  "weeklySchedule": [{ "dayOfWeek": 0, "startTime": "08:00", "endTime": "17:00", "isActive": true }, ...],
  "blockedDates": ["2026-03-05", "2026-03-12"],
  "existingBookings": [{ "date": "2026-03-10", "time": "14:00" }]
}
```
- No auth required (public calendar data for booking)

**AvailabilityCalendar (`src/features/booking/components/AvailabilityCalendar.tsx`):**

"use client" component. Props: `providerId: string`, `onDateSelect: (date: Date) => void`, `selectedDate?: Date`.

- Renders a month calendar grid (Mon-Sun headers, date cells)
- Fetches availability via `/api/provider/availability?providerId=X&month=YYYY-MM`
- Navigation arrows for prev/next month (don't allow past months)
- Visually disable (gray out) days where:
  - Provider has isActive=false for that dayOfWeek
  - Date is in blockedDates array
  - Date is in the past
- Available dates are clickable, highlighted on selection
- Use Tailwind for styling: rounded cells, primary color for selected, muted for disabled
- Mobile-first: calendar fits in mobile width

**TimeSlotPicker (`src/features/booking/components/TimeSlotPicker.tsx`):**

"use client" component. Props: `startTime: string`, `endTime: string`, `durationMinutes: number`, `existingBookings: string[]`, `onTimeSelect: (time: string) => void`, `selectedTime?: string`.

- Gray out slots that overlap with existingBookings
- Render as a grid of time buttons (3-4 columns on mobile, responsive)
- Selected slot gets primary color highlight

**BookingWizard (`src/features/booking/components/BookingWizard.tsx`):**

"use client" component. Props: `service` (id, title, fixedPrice, durationMinutes, pricingType, photoUrls, provider: { id, displayName, photoUrl, availabilities, blockedDates }).

3-step wizard using useState for step tracking (1, 2, 3):

**Step 1 — Date & Time Selection:**
- Render AvailabilityCalendar (pass providerId)
- Once date selected, render TimeSlotPicker below (pass provider hours for selected day)
- "Suivant" button enabled only when both date and time are selected
- Show selected service summary card (title, price, duration, provider name)

**Step 2 — Address & Details:**
- Form fields: address (Input, required), city (Input, required), clientNote (Textarea, optional)
- Pre-fill city if possible from service provider's delegation
- "Suivant" button, "Retour" button
- Validate with createBookingSchema partial (address + city fields)

**Step 3 — Confirmation & Payment:**
- Show full booking summary: service name, provider, date/time, address, price, duration
- Render PaymentMethodSelector
- "Mode demonstration" Alert banner at top explaining simulated payment

- On success: redirect to booking detail page (/bookings/[bookingId])
- On error: show error toast
- "Retour" button to go back

Step indicator at top: numbered circles (1, 2, 3) with connecting lines, active step highlighted.
