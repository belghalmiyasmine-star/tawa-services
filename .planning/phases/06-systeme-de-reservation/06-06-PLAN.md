**StatusTimeline (`src/features/booking/components/StatusTimeline.tsx`):**

Reusable component (used by both client and provider detail pages). Props: `booking` (status, createdAt, scheduledAt, completedAt, cancelledAt, cancelledBy).

Renders a vertical timeline showing booking progression:
- Step 1: "Reservation creee" — always shown, timestamp = createdAt
- Step 2: "Acceptee par le prestataire" — shown if status >= ACCEPTED
- Step 3: "Service en cours" — shown if status >= IN_PROGRESS
- Step 4: "Service termine" — shown if status = COMPLETED, timestamp = completedAt

Alternative paths:
- "Rejetee" — if status = REJECTED
- "Annulee" — if status = CANCELLED, show cancelledBy ("par le client" / "par le prestataire") and cancelledAt

Visual design:
- Vertical line connecting steps
- Active/completed steps: primary color dot + solid line
- Current step: animated pulse dot
- Future steps: muted gray dot + dashed line
- Cancelled/rejected: red dot at terminal step
- Each step shows formatted timestamp if available

Mobile-optimized: left-aligned timeline with dots on left, content on right.

**CancelBookingDialog (`src/features/booking/components/CancelBookingDialog.tsx`):**

"use client" component. Props: `bookingId: string`, `scheduledAt: Date`, `totalAmount: number`, `onCancelled: () => void`.

Uses shadcn AlertDialog:
- Title: "Annuler la reservation"
- Content:
  - Calculate refund tier using calculateRefundPercentage(scheduledAt) from cancellation utility (call the pure function client-side — import it)
  - Display prominent refund info card:
    - If FULL (>48h): green background — "Remboursement integral: {amount} TND"
    - If PARTIAL (24-48h): amber background — "Remboursement partiel: {amount * 0.5} TND ({hoursUntilScheduled}h avant le rendez-vous)"
    - If NONE (<24h): red background — "Aucun remboursement ({hoursUntilScheduled}h avant le rendez-vous)"
  - Reason textarea (optional): "Raison de l'annulation (optionnel)"
  - Warning text: "Cette action est irreversible"
- Cancel button: "Garder la reservation"
- Confirm button: "Confirmer l'annulation" (red variant)
  - On click: call cancelBookingAction({ bookingId, reason })
  - Loading state on confirm button
  - On success: toast + call onCancelled()
  - On error: error toast
