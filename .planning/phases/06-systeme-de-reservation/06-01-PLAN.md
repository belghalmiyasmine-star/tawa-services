**Booking actions (`src/features/booking/actions/manage-bookings.ts`):**
All actions use ActionResult<T> pattern, validate session via getServerSession, parse input with Zod schemas.

1. `createBookingAction(data)`:
   - Validates CLIENT role session
   - Parses with createBookingSchema
   - Verifies service exists, is ACTIVE, is not client's own service
   - Verifies provider availability: check Availability for scheduledAt dayOfWeek is active, time within startTime-endTime; check BlockedDate for date not blocked; check no existing PENDING/ACCEPTED booking for same service+scheduledAt date
   - Creates Booking with status PENDING, totalAmount = service.fixedPrice (or 0 for SUR_DEVIS which should not use direct booking)
   - Creates Payment with status PENDING, method from input, amount = totalAmount
   - Returns `{ bookingId }`

2. `acceptBookingAction(bookingId)`:
   - Validates PROVIDER role session
   - Verifies booking belongs to provider (via service.provider.userId)
   - Verifies status is PENDING
   - Updates status to ACCEPTED
   - Returns `{ success: true }`

3. `rejectBookingAction(data)`:
   - Parses with rejectBookingSchema
   - Validates PROVIDER role, ownership, status PENDING
   - Updates status to REJECTED, sets providerNote = reason
   - Returns `{ success: true }`

4. `startBookingAction(bookingId)`:
   - Validates PROVIDER role, ownership, status ACCEPTED
   - Updates status to IN_PROGRESS
   - Returns `{ success: true }`

5. `completeBookingAction(bookingId)`:
   - Validates PROVIDER role, ownership, status IN_PROGRESS
   - Updates status to COMPLETED, completedAt = now()
   - Updates Payment status from PENDING to HELD, heldAt = now()
   - Increments provider.completedMissions by 1
   - Returns `{ success: true }`

**Quote actions (`src/features/booking/actions/manage-quotes.ts`):**

1. `createQuoteAction(data)`:
   - Validates CLIENT role session
   - Parses with createQuoteSchema
   - Verifies service exists, is ACTIVE, pricingType is SUR_DEVIS
   - Creates Quote with status PENDING, expiresAt = now() + 48h, client fields (address, city, preferredDate, budget)
   - Returns `{ quoteId }`

2. `respondQuoteAction(data)`:
   - Validates PROVIDER role session
   - Parses with respondQuoteSchema
   - Verifies quote belongs to provider (via service.provider.userId), status is PENDING, not expired
   - Updates quote: status RESPONDED, proposedPrice, proposedDelay, respondedAt = now()
   - Returns `{ success: true }`

3. `acceptQuoteAction(data)`:
   - Validates CLIENT role session
   - Parses with acceptQuoteSchema
   - Verifies quote belongs to client (clientId), status is RESPONDED
   - In a transaction: update quote status to ACCEPTED + acceptedAt = now(); create Booking linked to quote (quoteId field), status PENDING, totalAmount = proposedPrice, scheduledAt from input; create Payment with status PENDING
   - Returns `{ bookingId }`

4. `declineQuoteAction(quoteId)`:
   - Validates CLIENT role session
   - Verifies quote belongs to client, status is RESPONDED
   - Updates status to DECLINED
   - Returns `{ success: true }`

**Query actions (`src/features/booking/actions/booking-queries.ts`):**

1. `getClientBookingsAction(filters?)`:
   - Validates CLIENT role session
   - Queries bookings where clientId = user.id, isDeleted false
   - Optional status filter (array of BookingStatus)
   - Includes service (title, photoUrls[0], pricingType, fixedPrice), provider (displayName, photoUrl)
   - Orders by createdAt desc
   - Returns paginated results (page, limit, total, bookings[])

2. `getProviderBookingsAction(filters?)`:
   - Validates PROVIDER role session
   - Queries bookings where provider.userId = session user id, isDeleted false
   - Optional status filter
   - Includes service (title, photoUrls[0]), client (firstName, lastName)
   - Orders by createdAt desc
   - Returns paginated results

3. `getBookingDetailAction(bookingId)`:
   - Validates session (CLIENT or PROVIDER)
   - Verifies booking belongs to user (clientId or provider.userId)
   - Includes full relations: service (all fields), provider (displayName, photoUrl, phone, rating), client (firstName, lastName, email), payment, quote
   - Returns booking detail object

4. `getProviderQuotesAction(filters?)`:
   - Validates PROVIDER role session
   - Queries quotes where service.provider.userId = session user id, isDeleted false
   - Optional status filter
   - Includes service (title), client info via clientId join
   - Orders by createdAt desc
   - Returns paginated results

Use existing patterns: ActionResult<T>, getServerSession(authOptions), prisma from @/lib/prisma, Zod parse with safeParse for validation errors.
