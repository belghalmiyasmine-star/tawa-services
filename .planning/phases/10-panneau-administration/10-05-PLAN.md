2. Create src/features/admin/actions/analytics-queries.ts ("use server"):
- All queries check ADMIN role via getServerSession.
- getAnalyticsDataAction(startDate?: string, endDate?: string): Returns object with:
  a) KPIs:
     - activeUsers: count of users where isActive=true AND createdAt > 30 days ago OR has any booking in last 30 days
     - totalTransactions: count of payments with status RELEASED in date range
     - totalRevenue: sum of payment.amount where status=RELEASED in date range
     - conversionRate: (bookings with status COMPLETED / total bookings) * 100, in date range. Target: 5%.
     - satisfactionRate: average of review.stars where published=true, in date range. Format as percentage out of 5.
     - avgProviderValidation: average time between provider.createdAt and provider KYC approval (kycStatus change to APPROVED). Use a reasonable approximation — count of providers with APPROVED KYC where the time difference is tracked. Target: <48h.
  b) Monthly series (last 6 or 12 months depending on range):
     - revenueByMonth: [{ month: "2026-01", revenue: number }]
     - bookingsByMonth: [{ month: "2026-01", count: number }]
     - userGrowthByMonth: [{ month: "2026-01", newUsers: number }]
  c) Breakdown data:
     - revenueByCategory: [{ category: string, revenue: number }] top 10
     - bookingsByStatus: { PENDING: n, ACCEPTED: n, COMPLETED: n, CANCELLED: n, REJECTED: n }
  Use in-memory groupBy for monthly data (Prisma findMany with date filter, then JS reduce to group by month). Dates formatted as "YYYY-MM".
  If no startDate/endDate provided, default to last 6 months.

- getGeographicBreakdownAction(startDate?: string, endDate?: string): Returns [{ city: string, bookings: number, revenue: number }] from provider delegations joined with bookings. Top 10 cities.

- getTopCategoriesAction(startDate?: string, endDate?: string): Returns [{ category: string, services: number, bookings: number, revenue: number }] top 10 categories by booking count.
