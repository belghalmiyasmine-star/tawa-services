1. Create src/features/admin/lib/csv-generator.ts:
- generateCsv(columns: { key: string, label: string }[], data: Record<string, unknown>[]): string
- Generates RFC 4180 compliant CSV:
  - Header row from column labels
  - Data rows: escape values with double quotes if they contain commas, newlines, or quotes
  - Double-quote escaping: replace " with ""
  - UTF-8 BOM prepended (\uFEFF) for Excel compatibility with French characters
- No external library needed — pure string manipulation.

2. Create src/features/admin/lib/pdf-generator.ts:
- generatePdfHtml(title: string, columns: { key: string, label: string }[], data: Record<string, unknown>[], metadata?: { generatedAt: string, generatedBy: string, dateRange?: string }): string
- Generates a printable HTML document with:
  - DOCTYPE html with UTF-8 charset
  - Embedded CSS (no external deps): clean table layout, @media print styles
  - Header: Tawa Services logo text, title, generated date, date range if provided
  - Table: striped rows, borders, header row bold with background
  - Footer: "Genere par Tawa Services" + page numbers via @page CSS
- The HTML is served as Content-Type text/html with Content-Disposition inline — browser prints to PDF via window.print().
- Alternative: if the user wants direct PDF download, use a simple approach where the API returns HTML and the client opens it in a new window for printing. This avoids heavy PDF libraries.

3. Create src/features/admin/actions/export-actions.ts ("use server"):
- getExportDataAction(type: "users" | "services" | "transactions" | "revenue" | "reports", filters?: { startDate?: string, endDate?: string }):
  - Checks ADMIN role.
  - Based on type, queries full dataset (no pagination limit) from DB:
    - "users": all users with name, email, role, status, created, phone
    - "services": all services with title, provider, category, price, status, created
    - "transactions": all payments with booking ref, client, provider, amount, commission, status, date
    - "revenue": monthly revenue summary (month, revenue, commission, net)
    - "reports": all reports with reporter, type, priority, status, created, resolved
  - Returns { columns: { key, label }[], data: Record<string, unknown>[] }.
  - Columns use French labels from hardcoded map (not i18n — server-side export).

4. Create src/app/api/admin/export/route.ts:
- GET handler with searchParams: type (users/services/transactions/revenue/reports), format (csv/pdf), columns (comma-separated column keys, optional).
- Validate ADMIN session via getServerSession.
- Call getExportDataAction(type, filters).
- If columns param provided, filter to only requested columns.
- If format=csv:
  - Generate CSV via generateCsv.
  - Return Response with Content-Type: text/csv; charset=utf-8, Content-Disposition: attachment; filename="tawa-{type}-{date}.csv"
- If format=pdf:
  - Generate HTML via generatePdfHtml.
  - Return Response with Content-Type: text/html; charset=utf-8 (opens in new tab for printing).
- Handle errors: 401 for unauthenticated, 403 for non-admin, 400 for invalid params.
