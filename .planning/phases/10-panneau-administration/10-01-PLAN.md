1. Add new enums and models to prisma/schema.prisma AFTER the NotificationPreference model:

```
enum ReportPriority {
  CRITICAL    // SLA < 2h
  IMPORTANT   // SLA < 24h
  MINOR       // SLA < 48h
}

enum ReportStatus {
  OPEN
  INVESTIGATING
  RESOLVED
  DISMISSED
}

enum ReportType {
  USER
  SERVICE
  REVIEW
  MESSAGE
}

model Report {
  id            String         @id @default(cuid())
  reporterId    String
  reporter      User           @relation("ReporterReports", fields: [reporterId], references: [id])
  reportedId    String?        // userId of reported person (nullable for service/review reports)
  reported      User?          @relation("ReportedReports", fields: [reportedId], references: [id])
  type          ReportType
  reason        String         @db.Text
  description   String?        @db.Text
  priority      ReportPriority @default(MINOR)
  status        ReportStatus   @default(OPEN)
  adminNote     String?        @db.Text
  assignedTo    String?        // admin userId
  referenceId   String?        // ID of reported service/review/message
  slaDeadline   DateTime?      // computed from priority + createdAt
  resolvedAt    DateTime?
  createdAt     DateTime       @default(now())
  updatedAt     DateTime       @updatedAt
  isDeleted     Boolean        @default(false)
  deletedAt     DateTime?

  @@map("reports")
}

model Faq {
  id        String   @id @default(cuid())
  question  String   @db.Text
  answer    String   @db.Text
  category  String?  // "general", "booking", "payment", "provider"
  sortOrder Int      @default(0)
  isActive  Boolean  @default(true)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  isDeleted Boolean  @default(false)
  deletedAt DateTime?

  @@map("faqs")
}

model LegalPage {
  id        String   @id @default(cuid())
  slug      String   @unique  // "cgu", "privacy", "legal-mentions"
  title     String
  content   String   @db.Text
  updatedBy String?  // admin userId
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@map("legal_pages")
}

model Banner {
  id         String   @id @default(cuid())
  title      String
  subtitle   String?
  imageUrl   String?
  linkUrl    String?
  position   String   @default("homepage")  // "homepage", "search", "category"
  isActive   Boolean  @default(true)
  sortOrder  Int      @default(0)
  startDate  DateTime?
  endDate    DateTime?
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt
  isDeleted  Boolean  @default(false)
  deletedAt  DateTime?

  @@map("banners")
}
```

- banUserSchema: z.object({ userId: z.string(), reason: z.string().min(5).max(500) })
- unbanUserSchema: z.object({ userId: z.string() })
- approveServiceSchema: z.object({ serviceId: z.string() })
- suspendServiceSchema: z.object({ serviceId: z.string(), reason: z.string().min(5).max(500).optional() })
- createReportSchema: z.object({ reportedId: z.string().optional(), type: z.enum(["USER","SERVICE","REVIEW","MESSAGE"]), reason: z.string().min(10).max(2000), description: z.string().optional(), priority: z.enum(["CRITICAL","IMPORTANT","MINOR"]).default("MINOR"), referenceId: z.string().optional() })
- updateReportSchema: z.object({ reportId: z.string(), status: z.enum(["OPEN","INVESTIGATING","RESOLVED","DISMISSED"]), adminNote: z.string().max(2000).optional() })
- adminUserFilterSchema: z.object({ search: z.string().optional(), role: z.enum(["CLIENT","PROVIDER","ADMIN"]).optional(), status: z.enum(["active","banned","inactive"]).optional(), page: z.coerce.number().min(1).default(1), pageSize: z.coerce.number().min(5).max(100).default(20) })
- adminServiceFilterSchema: z.object({ search: z.string().optional(), status: z.enum(["DRAFT","PENDING_APPROVAL","ACTIVE","SUSPENDED","DELETED"]).optional(), categoryId: z.string().optional(), page: z.coerce.number().min(1).default(1), pageSize: z.coerce.number().min(5).max(100).default(20) })

4. Create src/features/admin/actions/admin-queries.ts ("use server"):
- getAdminUsersAction(filters): paginated user list with avatar, name, email, role, isActive, isBanned, createdAt. Uses adminUserFilterSchema. Include provider relation for KYC status.
- getAdminServicesAction(filters): paginated service list with title, provider name, category name, price, status, isFeatured. Uses adminServiceFilterSchema.
- getAdminReportsAction(filters): paginated report list with reporter name, reported name, type, reason, priority, status, slaDeadline, createdAt. Ordered by priority (CRITICAL first) then createdAt DESC.
- getAdminStatsAction(): returns { totalUsers, totalProviders, totalBookings, totalRevenue, pendingKyc, activeServices, openReports, monthlyRevenue[] (last 6 months), bookingsByStatus, userGrowth[] (last 6 months), revenueByCategory[] }. Uses raw Prisma queries with groupBy where needed.
- getUserDetailAction(userId): full user detail with provider profile, bookings count, reviews count, reports count.
- getReportDetailAction(reportId): full report with reporter, reported user, admin notes, reference details.

5. Create src/features/admin/actions/admin-actions.ts ("use server"):
- All actions check session.user.role === "ADMIN" first, return ActionResult<T>.
- banUserAction(data): validates banUserSchema, sets user.isBanned=true, user.bannedAt=now, user.bannedReason=reason. Cannot ban ADMIN users.
- unbanUserAction(data): validates unbanUserSchema, sets user.isBanned=false, user.bannedAt=null, user.bannedReason=null.
- activateUserAction(userId): sets user.isActive=true.
- deactivateUserAction(userId): sets user.isActive=false. Cannot deactivate ADMIN.
- approveServiceAction(data): validates, sets service.status=ACTIVE.
- suspendServiceAction(data): validates, sets service.status=SUSPENDED.
- updateReportAction(data): validates updateReportSchema, updates report status + adminNote. If RESOLVED/DISMISSED, sets resolvedAt=now. Computes slaDeadline on creation based on priority (CRITICAL: +2h, IMPORTANT: +24h, MINOR: +48h).
- createReportAction(data): validates createReportSchema, creates report with computed slaDeadline.
- deleteUserAction(userId): soft-delete (isDeleted=true, deletedAt=now). Cannot delete ADMIN.

Use the ActionResult<T> pattern from existing features (return { success: true, data } | { success: false, error: string }).
