All server actions follow the established `ActionResult<T>` pattern from `src/types/api.ts`. Each has `"use server"` directive, validates session via `getServerSession(authOptions)`, checks `role === "PROVIDER"`, finds provider record, validates input with Zod schema, performs Prisma operation, returns `ActionResult<T>`.

**`src/features/provider/actions/update-profile.ts`** â€” `updateProfileAction(data: UpdateProfileFormData)`:
- Parse with `updateProfileSchema`
- `prisma.provider.update({ where: { userId: session.user.id }, data: { displayName, bio, phone, yearsExperience, languages } })`
- Return `{ success: true, data: { id: provider.id } }`

**`src/features/provider/actions/manage-zones.ts`** â€” `updateZonesAction(data: ZoneFormData)`:
- Parse with `zoneSchema`
- Use `prisma.$transaction`:
  1. `prisma.providerDelegation.deleteMany({ where: { providerId } })` â€” clear all existing zones
  2. `prisma.providerDelegation.createMany({ data: data.delegationIds.map(delegationId => ({ providerId, delegationId })) })`
- Return `{ success: true, data: { count: data.delegationIds.length } }`

**`src/features/provider/actions/manage-availability.ts`** â€” two actions:

1. `updateAvailabilityAction(data: AvailabilityFormData)`:
   - Parse with `availabilitySchema`
   - Use `prisma.$transaction` with 7 upserts (one per day): `prisma.availability.upsert({ where: { providerId_dayOfWeek: { providerId, dayOfWeek } }, create: { providerId, dayOfWeek, startTime, endTime, isActive }, update: { startTime, endTime, isActive } })`
   - Return `{ success: true, data: { updated: 7 } }`

2. `updateBlockedDatesAction(data: BlockedDateFormData)`:
   - Parse with `blockedDateSchema`
   - Use `prisma.$transaction`:
     1. Delete all existing blocked dates for provider (replace strategy)
     2. `prisma.blockedDate.createMany({ data: dates.map(d => ({ providerId, date: new Date(d.date), reason: d.reason })) })`
   - Return `{ success: true, data: { count: dates.length } }`

**`src/features/provider/actions/manage-portfolio.ts`** â€” `"use server"`, three actions:

1. `getPortfolioPhotosAction()`:
   - Fetch all non-deleted portfolio photos for current provider
   - `prisma.portfolioPhoto.findMany({ where: { providerId, isDeleted: false }, orderBy: { sortOrder: "asc" } })`
   - Return `ActionResult` with array

2. `updatePortfolioPhotoCaptionAction(data: { photoId: string, caption: string })`:
   - Verify ownership (portfolioPhoto.providerId matches session provider)
   - Validate caption with portfolioPhotoSchema
   - `prisma.portfolioPhoto.update({ where: { id: photoId }, data: { caption } })`
   - Return `ActionResult`

3. `deletePortfolioPhotoAction(photoId: string)`:
   - Verify ownership
   - Soft delete: `prisma.portfolioPhoto.update({ where: { id: photoId }, data: { isDeleted: true, deletedAt: new Date() } })`
   - Delete physical file from disk (best effort)
   - Return `ActionResult`

**`src/app/api/provider/photo/route.ts`** â€” POST handler:
- Follow same pattern as `src/app/api/kyc/upload/route.ts` (multipart form data, auth check, MIME validation)
- Accept `file` field (image/jpeg, image/png, image/webp), max 5MB
- Store to `/public/uploads/providers/[userId]/avatar-[uuid].[ext]`
- After saving, update `prisma.provider.update({ where: { userId }, data: { photoUrl: relativePath } })`
- Return `{ success: true, data: { photoUrl } }`

**`src/app/api/provider/portfolio/route.ts`** â€” POST and DELETE handlers:

POST handler (upload portfolio photo):
- Follow same pattern as `src/app/api/kyc/upload/route.ts`
- Accept multipart form data: `file` (image/jpeg, image/png, image/webp), `caption` (string, optional, max 200 chars)
- Auth check: 401 no session, 403 not PROVIDER
- Check current portfolio count: if `portfolioPhoto count >= 10`, return 400 "Maximum 10 photos de portfolio"
- Store to `/public/uploads/providers/[userId]/portfolio/[uuid].[ext]`
- Determine next sortOrder: max existing sortOrder + 1
- Create DB record: `prisma.portfolioPhoto.create({ data: { providerId, photoUrl: relativeUrl, caption, sortOrder } })`
- Return `{ success: true, data: { id: photo.id, photoUrl: relativeUrl } }`

DELETE handler (remove portfolio photo):
- Accept JSON body: `{ photoId: string }`
- Auth + ownership check
- Soft delete record, delete physical file (best effort)
- Return `{ success: true, data: { removed: photoId } }`
