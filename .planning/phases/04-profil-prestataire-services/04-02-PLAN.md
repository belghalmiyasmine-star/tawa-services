   - Parse with createServiceSchema

**`src/app/api/service/photos/route.ts`** — POST and DELETE handlers:

POST handler (upload service work photo):
- Follow same pattern as `src/app/api/kyc/upload/route.ts`
- Accept multipart form data: `file` (image/jpeg, image/png, image/webp), `serviceId` (string)
- Auth check: 401 no session, 403 not PROVIDER
- Verify service ownership: `prisma.service.findFirst({ where: { id: serviceId, provider: { userId }, isDeleted: false } })`
- Check current photo count: if `service.photoUrls.length >= 5`, return 400 "Maximum 5 photos par service"
- Store to `/public/uploads/services/[serviceId]/[uuid].[ext]`
- Update service: `prisma.service.update({ where: { id: serviceId }, data: { photoUrls: { push: relativeUrl } } })`
- Return `{ success: true, data: { photoUrl: relativeUrl, totalPhotos: newCount } }`

DELETE handler (remove service photo):
- Accept JSON body: `{ serviceId: string, photoUrl: string }`
- Auth + ownership check
- Remove from array: `prisma.service.update({ where: { id: serviceId }, data: { photoUrls: service.photoUrls.filter(url => url !== photoUrl) } })`
- Delete physical file from disk using `fs.unlink` (best effort, don't fail if file missing)
- Return `{ success: true, data: { removed: photoUrl } }`

**`src/app/api/provider/certification/route.ts`** — POST handler:
- Accept multipart form data: `file` (image/jpeg, image/png, image/webp, application/pdf), `title` (string)
- Max size 10MB (certificates can be PDF)
- Auth + PROVIDER role check
- Store to `/public/uploads/providers/[userId]/certs/[uuid].[ext]`
- Create DB record: `prisma.certification.create({ data: { providerId, title, fileUrl: relativeUrl } })`
- Return `{ success: true, data: { id: cert.id, fileUrl: relativeUrl } }`
