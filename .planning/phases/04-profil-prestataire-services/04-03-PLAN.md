**`src/features/provider/components/PhotoUpload.tsx`** — Client component ("use client"):
- Props: `currentPhotoUrl?: string | null`, `onUploadComplete: (url: string) => void`
- Displays current photo in a rounded circle (128x128) or a placeholder avatar icon (lucide-react `UserCircle`)
- Click triggers hidden file input (accept="image/jpeg,image/png,image/webp")
- On file select: validate size (max 5MB client-side), show loading spinner, POST to `/api/provider/photo` with FormData
- On success: call `onUploadComplete(photoUrl)`, show toast success
- On error: show toast error with translated message
- Hover overlay shows camera icon (lucide-react `Camera`)
- Rounded Airbnb style per CONTEXT.md

**`src/features/provider/components/ZoneSelector.tsx`** — Client component ("use client"):
- Props: `initialDelegationIds: string[]`, `gouvernorats: Array<{ id: string, name: string, delegations: Array<{ id: string, name: string }> }>`
- Two-level selection: first select gouvernorats (checkboxes or accordion), then delegations within each expand
- Selected delegations shown as badge chips below the selector with X to remove
- "Tout selectionner" / "Tout deselectionner" per gouvernorat
- On save button click: calls `updateZonesAction({ delegationIds })` and shows toast
- Uses useTranslations("provider") for all labels
- Mobile-friendly: accordion collapse gouvernorats on mobile, columns on desktop

**`src/features/provider/components/PortfolioUploader.tsx`** — Client component ("use client"):
- Props: `initialPhotos: Array<{ id: string, photoUrl: string, caption: string | null, sortOrder: number }>`, `providerId: string`
- Grid display: 2 cols on mobile, 3 cols on desktop, up to 10 slots
- Each photo slot: 1:1 aspect ratio card showing image thumbnail or empty upload placeholder (dashed border + plus icon)
- Click empty slot → upload via POST `/api/provider/portfolio` with FormData (file + optional caption)
- Each filled slot has: image preview, editable caption text field below (max 200 chars), X button overlay to delete
- Delete calls DELETE `/api/provider/portfolio` with `{ photoId }` and removes from UI
- Caption edit: inline text input below each photo, on blur calls `updatePortfolioPhotoCaptionAction({ photoId, caption })`
- Counter badge: "3/10 photos"
- Show loading spinner per slot during upload
- Error toast if file invalid (not image, >5MB) or all 10 slots full
- All labels from useTranslations("provider") — keys: portfolioTitle, portfolioUpload, portfolioCaption, portfolioMax, portfolioCount, portfolioEmpty

- Uses react-hook-form with `zodResolver(updateProfileSchema)` — follow same pattern as RegisterWizard from Phase 2

  - `bio` — Textarea with character counter (X/2000)
  - `phone` — Input with +216 prefix hint
  - `yearsExperience` — Input type=number (0-50)
  - `languages` — Multi-select checkboxes: Francais, Arabe, Anglais, Amazigh, Espagnol, Italien, Allemand (common in Tunisia)

- All labels from useTranslations("provider")
- Layout: single column on mobile, two columns on desktop for shorter fields

**`src/app/[locale]/(provider)/provider/profile/edit/page.tsx`** — Server component:
- Auth check via `getServerSession(authOptions)` — redirect if not authenticated
- Fetch provider data: `prisma.provider.findUnique({ where: { userId }, include: { delegations: { include: { delegation: { include: { gouvernorat: true } } } } } })`
- Fetch all gouvernorats with delegations: `prisma.gouvernorat.findMany({ include: { delegations: true }, orderBy: { name: "asc" } })`
- Fetch provider availabilities: `prisma.availability.findMany({ where: { providerId } })`
- Fetch blocked dates: `prisma.blockedDate.findMany({ where: { providerId } })`
- Fetch portfolio photos: `prisma.portfolioPhoto.findMany({ where: { providerId, isDeleted: false }, orderBy: { sortOrder: "asc" } })`
- If no provider record exists, redirect to `/provider/kyc` — provider record is created during registration
- Render page with: heading (useTranslations), ProfileEditForm (pass provider data as defaultValues), ZoneSelector (pass gouvernorats list + initialDelegationIds), AvailabilityEditor (pass initial slots), BlockedDatesEditor (pass initial dates), PortfolioUploader (pass initial photos)
- Use Tabs component (shadcn) for sections: "Informations" | "Zones" | "Disponibilites" | "Portfolio" | "Certifications" — 5 tabs total (Certifications tab added by Plan 05)
- generateMetadata for SEO: title from t("editProfile")
