- Props: `serviceId: string | null` (null for new service — upload after creation), `initialPhotos: string[]`, `onPhotosChange: (urls: string[]) => void`
- Grid display: 2 cols on mobile, 3 cols on desktop, max 5 slots
- Each photo slot: 1:1 aspect ratio card showing image thumbnail or empty upload placeholder (dashed border + plus icon)
- Click empty slot or drag file onto it → upload via POST `/api/service/photos` with serviceId
- Each filled slot has: image preview, X button overlay to delete (DELETE `/api/service/photos`), drag handle for reorder
- Drag to reorder: use HTML5 native drag events (onDragStart, onDragOver, onDrop) — same pattern as KYC wizard per [03-02] decision. After reorder, update photoUrls order via updateServiceAction.
- Counter badge: "3/5 photos"
- Show loading spinner per slot during upload
- Error toast if file invalid (not image, >5MB) or slot full
- When serviceId is null (new service mode): disable uploads, show message "Enregistrez d'abord le service pour ajouter des photos"

  - `categoryId` — Two-level Select: first select main category (parentId=null), then sub-category filtered by parent. If no subcategories, single level. Use shadcn Select or Combobox.

  - `inclusions` — InclusionsExclusionsEditor type="inclusions"
  - `exclusions` — InclusionsExclusionsEditor type="exclusions"
  - `conditions` — Textarea (max 2000 chars), optional

- On create success: redirect to `/provider/services/[newId]/edit` so user can add photos
- On edit success: show toast "Service mis a jour"
- All labels from useTranslations("service")
- Layout: mobile single column, desktop two columns for price/duration row
