1. Create `src/features/payment/actions/earnings-queries.ts`:
   - "use server" directive
   - Import getServerSession, authOptions, prisma, ActionResult

   - `getProviderEarningsAction(): Promise<ActionResult<EarningsSummary>>`:
     - Validate PROVIDER session
     - Fetch provider by userId
     - Query payments where booking.providerId = provider.id:
       - `available`: SUM(providerEarning) where status=RELEASED and no withdrawalRequest
       - `pending`: SUM(amount) where status=HELD
       - `totalEarned`: SUM(providerEarning) where status=RELEASED
       - `totalCommission`: SUM(commission) where status=RELEASED
     - Use prisma.payment.aggregate with appropriate where clauses
     - Return { available, pending, totalEarned, totalCommission }

   - `getMonthlyBreakdownAction(): Promise<ActionResult<MonthlyBreakdown[]>>`:
     - Validate PROVIDER session
     - Query all RELEASED payments for provider's bookings
     - Group by month (extract year-month from releasedAt)
     - For each month: count missions, sum amount (gross), sum commission, sum providerEarning (net)
     - Use raw groupBy or in-memory grouping from findMany with select
     - Sort by month descending (newest first)
     - Return array of { month: string (YYYY-MM), missions: number, grossRevenue: number, commission: number, netEarnings: number }

   - `getTransactionHistoryAction(): Promise<ActionResult<TransactionItem[]>>`:
     - Validate PROVIDER session
     - Query payments for provider's bookings with status in [HELD, RELEASED, REFUNDED]
     - Include booking with service (title) and client (name)
     - Order by createdAt desc
     - Return array of { id, bookingId, serviceTitle, clientName, amount, commission, providerEarning, status, method, createdAt, releasedAt }

   - Define TypeScript interfaces for EarningsSummary, MonthlyBreakdown, TransactionItem at top of file

2. Create `src/features/payment/actions/withdrawal-actions.ts`:
   - "use server" directive

   - `requestWithdrawalAction(amount: number): Promise<ActionResult<{ requestId: string }>>`:
     - Validate PROVIDER session
     - Fetch provider by userId
     - Validate amount >= 50 (minimum 50 TND)
     - Query available balance: SUM(providerEarning) from RELEASED payments that have NO existing withdrawalRequest
     - If available < amount, return error "Solde insuffisant"
     - Find RELEASED payments without withdrawalRequest, ordered by releasedAt asc (FIFO)
     - Create WithdrawalRequest records linking to the specific payment(s) that cover the amount
     - NOTE: WithdrawalRequest schema has paymentId (unique, one-to-one) — so create one WithdrawalRequest per payment consumed, each with the portion amount. OR simpler approach: pick the first available RELEASED payment and create a single withdrawal request for `amount` up to that payment's providerEarning. For simplicity in PFE, create a single WithdrawalRequest linked to the oldest available payment, with amount = requested amount.
     - Return { requestId }

   - `getWithdrawalRequestsAction(): Promise<ActionResult<WithdrawalRequestItem[]>>`:
     - Validate PROVIDER session
     - Query WithdrawalRequest where payment.booking.providerId matches
     - Return list with id, amount, status, requestedAt, processedAt
