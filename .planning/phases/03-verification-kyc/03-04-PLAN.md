        - `QUICK_RESPONSE`: Green `Badge` with `Zap` icon (lucide), text "Reponse Rapide"

   - Each badge wrapped in a tooltip (use native `title` attribute or shadcn `Tooltip` if already installed):
     - IDENTITY_VERIFIED: "Ce prestataire a verifie son identite aupres de Tawa Services"
     - QUICK_RESPONSE: "Ce prestataire repond en moyenne en moins d'une heure"
     - TOP_PROVIDER: "Note superieure a 4.5 avec plus de 10 missions terminees"
   - Size variants: sm = `text-xs px-2 py-0.5`, md = `text-sm px-3 py-1`
   - Badges displayed in a horizontal flex row with `gap-1.5`

   - Export `TrustBadges` component and `TrustBadgesProps` type

2. Create `src/features/kyc/actions/compute-badges.ts` with `"use server"` directive:
   - Export `computeAndAwardBadges(providerId: string): Promise<void>`:
     - Find provider with `select: { id: true, kycStatus: true, rating: true, ratingCount: true, completedMissions: true, responseTimeHours: true }`
     - If provider not found, return silently (no error — defensive)
     - Badge computation rules (per CONTEXT.md):
       a. `IDENTITY_VERIFIED`: Already handled by approveKycAction — skip here (only compute auto badges)
       b. `QUICK_RESPONSE`: `provider.responseTimeHours !== null && provider.responseTimeHours < 1`
       c. `TOP_PROVIDER`: `provider.rating > 4.5 && provider.completedMissions > 10`
     - For each auto badge, use `prisma.trustBadge.upsert`:
       - `where: { providerId_badgeType: { providerId, badgeType } }`
       - `update: { isActive: shouldHaveBadge }` (set to true or false based on current stats)
       - `create: { providerId, badgeType, isActive: shouldHaveBadge }`
     - This function will be called after booking completion (Phase 6+) and rating updates (Phase 8+)
     - For now, it's callable but won't award QUICK_RESPONSE or TOP_PROVIDER since no bookings exist yet — this is fine

   - Export `getProviderBadges(providerId: string): Promise<{ badgeType: string; isActive: boolean }[]>`:
     - Simple query: `prisma.trustBadge.findMany({ where: { providerId, isActive: true } })`
     - Return array of `{ badgeType, isActive }`
     - This is a helper for use in server components that render provider profiles/cards

   Note: These are auto-computed badges. IDENTITY_VERIFIED is created by approveKycAction in Plan 03.
   The compute function handles QUICK_RESPONSE and TOP_PROVIDER only.
