  <action>
1. Create `src/lib/validations/kyc.ts`:
   - Export `KYC_DOC_TYPES` constant: `["CIN_RECTO", "CIN_VERSO", "SELFIE", "PROOF_OF_ADDRESS"]` (4-step per CONTEXT.md — NOT 3 as in ROADMAP)
   - Export `ALLOWED_MIME_TYPES`: `["image/jpeg", "image/jpg", "image/png"]`
   - Export `MAX_FILE_SIZE`: `5 * 1024 * 1024` (5MB)
   - Export `kycUploadSchema` (Zod): validates `docType` is one of KYC_DOC_TYPES
   - Export `kycSubmissionSchema` (Zod): validates all 4 document types are present (array of {docType, fileUrl})
   - Export TypeScript types: `KycDocType`, `KycUploadData`, `KycSubmissionData`

2. Create `src/app/api/kyc/upload/route.ts`:
   - POST handler accepting `multipart/form-data` with `file` (File) and `docType` (string) fields
   - Validate authenticated session via `getServerSession(authOptions)` — return 401 if not
   - Validate role is PROVIDER — return 403 if not
   - Validate `docType` against `KYC_DOC_TYPES` — return 400 if invalid
   - Validate file MIME type against `ALLOWED_MIME_TYPES` — return 400 with descriptive error
   - Validate file size <= `MAX_FILE_SIZE` — return 400 with descriptive error
   - Get provider record via `prisma.provider.findUnique({ where: { userId } })`
   - Generate UUID filename: `${crypto.randomUUID()}.${extension}`
   - Create directory `/public/uploads/kyc/${userId}/` if not exists (use `fs.mkdir` with `{ recursive: true }`)
   - Write file to disk using `fs.writeFile`
   - Return JSON `{ success: true, data: { fileUrl: "/uploads/kyc/${userId}/${filename}", docType } }`
   - Wrap in try/catch, return 500 on failure
   - Use Node.js `fs/promises` and `path` modules
   - Import `NextResponse` from `next/server`
  </action>
  <verify>
    - `npx tsc --noEmit` passes without errors
    - File `src/app/api/kyc/upload/route.ts` exports POST
    - File `src/lib/validations/kyc.ts` exports kycUploadSchema, kycSubmissionSchema, KYC_DOC_TYPES
  </verify>
  <done>Upload API accepts multipart files, validates type/size/auth, stores to disk, returns fileUrl. Validation schemas enforce 4 document types.</done>

1. Create `src/features/kyc/actions/submit-kyc.ts`:
   - `"use server"` directive at top
   - Import `prisma` from `@/lib/prisma`, `getServerSession` from `next-auth`, `authOptions` from `@/lib/auth`
   - Import `kycSubmissionSchema` from `@/lib/validations/kyc` and `ActionResult` from `@/types/api`
   - Export `submitKycAction(data: unknown): Promise<ActionResult<{ providerId: string }>>`:
     a. Get session, return error if not authenticated or not PROVIDER role
     b. Parse data with `kycSubmissionSchema.safeParse(data)` — return first Zod error if invalid
     c. Find provider by `userId` — return error if provider record not found
     d. Check provider.kycStatus — if PENDING or APPROVED, return error "Vous avez deja soumis vos documents"
     e. Use `prisma.$transaction` to atomically:
        - Delete any existing KYCDocument records for this provider (in case of re-submission after REJECTED)
        - Create 4 KYCDocument records (one per docType from parsed data) with `providerId`, `docType`, `fileUrl`
        - Update provider: `kycStatus: "PENDING"`, `kycSubmittedAt: new Date()`
        - If kycStatus was REJECTED, clear `kycRejectedAt` and `kycRejectedReason`
     f. Return `{ success: true, data: { providerId: provider.id } }`
     g. Catch errors, log, return generic error message
   - Follow same pattern as `src/features/auth/actions/register.ts` (ActionResult, Zod parsing, try/catch)
