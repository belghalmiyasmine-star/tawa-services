  <name>Task 1: Create admin KYC review server actions</name>
  <files>
    src/features/kyc/actions/review-kyc.ts
  </files>
  <action>
1. Create `src/features/kyc/actions/review-kyc.ts` with `"use server"` directive:

   a. Export `approveKycAction(providerId: string): Promise<ActionResult<{ providerId: string }>>`:
      - Get session, verify ADMIN role — return error if not
      - Find provider by id, verify kycStatus is PENDING — return error if not
      - Use `prisma.$transaction` to atomically:
        - Update provider: `kycStatus: "APPROVED"`, `kycApprovedAt: new Date()`, clear `kycRejectedAt` and `kycRejectedReason`
        - Create TrustBadge record: `{ providerId, badgeType: "IDENTITY_VERIFIED", isActive: true }` (use upsert to avoid duplicate constraint violation)
        - Create Notification: `{ userId: provider.userId, type: "KYC_APPROVED", title: "Votre identite a ete verifiee", body: "Felicitations ! Vous pouvez maintenant proposer vos services." }`
      - Return success with providerId
      - Note: CONTEXT.md says "sets user.isVerified=true" but schema uses provider.kycStatus — use provider.kycStatus per schema (no isVerified field on User)

   b. Export `rejectKycAction(data: { providerId: string; reason: string; comment?: string }): Promise<ActionResult<{ providerId: string }>>`:
      - Get session, verify ADMIN role
      - Validate reason is one of: "illegible", "nonConforming", "mismatch", "other"
      - Find provider by id, verify kycStatus is PENDING
      - Use `prisma.$transaction`:
        - Update provider: `kycStatus: "REJECTED"`, `kycRejectedAt: new Date()`, `kycRejectedReason: reason + (comment ? ": " + comment : "")`
        - Create Notification: `{ userId: provider.userId, type: "KYC_REJECTED", title: "Verification d'identite rejetee", body: "Votre dossier a ete rejete. Motif: ..." }`
      - Return success
      - Combine dropdown reason with optional comment into kycRejectedReason field

   c. Export `getKycSubmissions(): Promise<ActionResult<KycSubmissionSummary[]>>` (helper for list page):
      - Get session, verify ADMIN role
      - Query providers with `kycStatus: "PENDING"` ordered by `kycSubmittedAt ASC` (oldest first)
      - Include `user` relation (name, email, phone, createdAt) and `kycDocuments`
      - For each submission, compute `isOverdue: boolean` = `kycSubmittedAt + 48h < now()`
      - Return array of `{ providerId, displayName, email, phone, registeredAt, submittedAt, documentCount, isOverdue }`
      - Define `KycSubmissionSummary` type in same file

   d. Export `getKycSubmissionDetail(providerId: string): Promise<ActionResult<KycSubmissionDetail>>`:
      - Get session, verify ADMIN role
      - Find provider with `include: { user: true, kycDocuments: true }`
      - Return `{ providerId, displayName, email, phone, registeredAt, submittedAt, kycStatus, documents: [{docType, fileUrl, uploadedAt}], isOverdue }`
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - All 4 functions export correctly
    - Approve creates IDENTITY_VERIFIED TrustBadge and KYC_APPROVED notification
    - Reject stores combined reason + comment in kycRejectedReason
  </verify>
  <done>Admin server actions for KYC review: approve (sets APPROVED + creates badge + notification), reject (sets REJECTED with reason + notification), list (pending submissions sorted oldest first with overdue flag), and detail (full provider info + documents).</done>
