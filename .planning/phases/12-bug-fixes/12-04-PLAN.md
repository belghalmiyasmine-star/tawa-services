    **BUGF-12: Auto-moderation regex catches phone numbers and emails**

    The existing moderation.ts has regex patterns but they may not catch all common Tunisian phone formats and obfuscated contact info.

    1. **Improve the regex patterns in `src/features/review/lib/moderation.ts`:**

       Current patterns look reasonable but add these additional patterns:
       - Spaces between digits: `2 0 1 2 3 4 5 6` (each digit separated)
       - Written-out numbers: "vingt" etc. — skip this, too complex
       - "at" instead of "@": `user at gmail dot com` pattern
       - Phone with parentheses: `(+216) 20 123 456`
       - Phone with slash separators: `20/123/456`

       Update PHONE_TN_REGEX to also catch:
       ```typescript
       const PHONE_TN_REGEX =
         /(\+?216[\s./()-]?\d{2}[\s./()-]?\d{3}[\s./()-]?\d{3}|\b\d{2}[\s./()-]?\d{3}[\s./()-]?\d{3}\b)/g;
       ```

       Add a pattern for spaced-out digits (anti-evasion):
       ```typescript
       const SPACED_DIGITS_REGEX = /\b\d[\s.,-]\d[\s.,-]\d[\s.,-]\d[\s.,-]\d[\s.,-]\d[\s.,-]\d[\s.,-]\d\b/g;
       ```

       Add "at/dot" email evasion pattern:
       ```typescript
       const EMAIL_OBFUSCATED_REGEX = /\b\w+\s*(at|chez)\s*\w+\s*(dot|point)\s*\w+\b/gi;
       ```

    2. **Add the new patterns to `detectContactInfo`** — push matches from SPACED_DIGITS_REGEX and EMAIL_OBFUSCATED_REGEX into the matches array.

    3. **Verify moderation is applied in messaging too.** Read `src/features/messaging/components/ChatView.tsx` — check if message sending calls `detectContactInfo` or `moderateReviewContent` before sending. If not:
       - Find the message send action (likely in `src/features/messaging/actions/`)
       - Import `detectContactInfo` from the moderation lib
       - Before saving the message, check for contact info. If detected, either:
         a) Block the message and return an error, OR
         b) Flag the message (add a `flagged` field) but still save it
       - The simpler approach: import and call `detectContactInfo` in the send-message server action. If `hasContact` is true, return `{ success: false, error: "Message contient des informations de contact (téléphone ou email). Veuillez utiliser le système de messagerie de la plateforme." }`

    Keep the existing working regex patterns — only ADD new ones. Don't remove EMAIL_REGEX or PHONE_TN_REGEX.
