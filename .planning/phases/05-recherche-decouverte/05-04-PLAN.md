  <action>
    Create `src/app/api/search/autocomplete/route.ts`:

    - GET handler, public (no auth)
    - Accept query param `q` (string, min 2 chars to trigger, max 100 chars)
    - Return empty results if `q` is less than 2 chars
    - Run two parallel Prisma queries using `Promise.all`:

      a. **Category matches**: `prisma.category.findMany` where name contains `q` (mode: insensitive), isActive: true, isDeleted: false. Take 3. Select: id, name, slug, icon.

      b. **Service matches**: `prisma.service.findMany` where `(title contains q OR description contains q)` AND status: ACTIVE, isDeleted: false. Take 5. Select: id, title, fixedPrice, pricingType, category: { name, slug }, provider: { displayName, photoUrl, rating, delegations (take 1, with delegation.gouvernorat.name) }.

    - Return JSON: `{ categories: [...], services: [...] }`
    - Performance: Keep queries lightweight — only use `contains` with `mode: insensitive`, limit results, select minimal fields
    - Wrap in try/catch, return 400 on invalid query, 500 on DB error
    - Set `Cache-Control: no-store` header (results depend on live DB data)
