<task type="auto">
  <name>Task 2: Create service search API endpoint with filters, sort, and pagination</name>
  <files>src/app/api/search/services/route.ts</files>
  <action>
    Create `src/app/api/search/services/route.ts`:

    - GET handler, public (no auth required)
    - Parse URL searchParams using `searchParamsSchema` from `@/lib/validations/search`
    - Build Prisma `where` clause dynamically:
      - Always: `status: "ACTIVE"`, `isDeleted: false`
      - `q`: Use Prisma `contains` (case-insensitive via `mode: "insensitive"`) on `title` OR `description` OR `provider.displayName`
      - `category`: Match by `category.slug` (exact match), OR if the slug matches a parent category, include all children via `category.parentId` lookup
      - `city`: Filter via `provider.delegations.some({ delegation: { gouvernorat: { name: city } } })`
      - `delegation`: Filter via `provider.delegations.some({ delegation: { name: delegation } })`
      - `minPrice`/`maxPrice`: Filter on `fixedPrice` (only for FIXED pricing). Services with SUR_DEVIS pass through if no price filter is set.
      - `pricingType`: Exact match on service pricingType
      - `verified`: If true, filter `provider.kycStatus: "APPROVED"`
      - `minRating`: Filter `provider.rating >= minRating`
    - Build Prisma `orderBy` from `sort` param:
      - `"relevance"`: `[{ isFeatured: "desc" }, { viewCount: "desc" }, { createdAt: "desc" }]`
      - `"price_asc"`: `{ fixedPrice: "asc" }` (nulls last — SUR_DEVIS services at end)
      - `"price_desc"`: `{ fixedPrice: "desc" }` (nulls last)
      - `"rating"`: `{ provider: { rating: "desc" } }`
      - `"newest"`: `{ createdAt: "desc" }`
    - Pagination: `skip: (page - 1) * limit`, `take: limit`
    - Include in each service result:
      - `category` (with `parent`)
      - `provider` (select: `id, displayName, photoUrl, rating, ratingCount, kycStatus, delegations` with delegation.gouvernorat)
    - Also run `prisma.service.count({ where })` for total count
    - Return JSON: `{ services: [...], total: number, page: number, limit: number, totalPages: Math.ceil(total / limit) }`
    - Wrap in try/catch, return 400 on validation errors, 500 on DB errors
  </action>
