**1. Create `src/features/review/lib/publication.ts`:**

- `publishBothReviews(bookingId: string): Promise<void>`
  - Find all reviews for this bookingId where isDeleted=false
  - If exactly 2 reviews exist (one CLIENT, one PROVIDER):
    - Set published=true, publishedAt=new Date() on both via prisma.review.updateMany
    - Extract the providerId from the booking
    - Call updateProviderRating(providerId) to recompute aggregates
  - If only 1 review: do nothing (wait for second)

- `publishSoloReviewIfExpired(bookingId: string): Promise<void>`
  - Called by cron when window expires with only 1 review submitted
  - Find reviews for booking where isDeleted=false
  - If exactly 1 review exists and booking.completedAt + 10 days < now:
    - Set published=true, publishedAt=new Date() on the solo review
    - Update provider rating if the solo review targets a provider

- `checkAndCloseExpiredWindows(): Promise<{ processed: number; published: number }>`
  - Find all COMPLETED bookings where:
    - completedAt is between 10 and 11 days ago (process once, not repeatedly)
    - Have at least 1 unpublished review (review where published=false, isDeleted=false)
  - For each: call publishSoloReviewIfExpired(bookingId)
  - Return count of processed bookings and published reviews
  - Uses Prisma raw query or aggregation to efficiently find candidates

- `isReviewWindowOpen(booking: { completedAt: Date | null; status: string }): { open: boolean; daysRemaining: number }`
  - Pure function: returns open=true if status=COMPLETED and completedAt + 10 days > now
  - daysRemaining = Math.ceil((completedAt + 10 days - now) / (24*60*60*1000))
  - If completedAt is null or status !== COMPLETED: open=false, daysRemaining=0
