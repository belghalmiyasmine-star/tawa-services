2. Update `src/lib/auth.ts` -- enhance the CredentialsProvider authorize callback:
   - Before password check: query user, check if `lockedUntil` is set and in the future. If locked, throw/return null with message "Compte temporairement verrouille. Reessayez dans X minutes."
   - On failed login: increment `failedLoginAttempts` on the user record. If attempts >= 8 (3 initial + 5 with CAPTCHA), set `lockedUntil = new Date(Date.now() + 15 * 60 * 1000)` and reset failedLoginAttempts to 0.
   - On successful login: reset `failedLoginAttempts` to 0 and clear `lockedUntil`.
   - The CAPTCHA verification itself happens client-side (LoginForm sends a captchaToken when attempts >= 3). In authorize, if failedLoginAttempts >= 3 and no captchaToken provided, return null with error "CAPTCHA requis".
   - For OAuth signIn callback: implement same-email auto-linking. If user with same email exists and the account record doesn't exist yet, create the Account record linking the OAuth provider to the existing user. Use prisma.account.create inside the signIn callback.
