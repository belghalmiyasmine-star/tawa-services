  <files>

  <action>
1. Rewrite `src/middleware.ts` to combine next-intl middleware with auth checks:
   - Import `getToken` from "next-auth/jwt" (reads JWT from request cookies without DB call)
   - Import `createMiddleware` from "next-intl/middleware" and `routing` from "@/i18n/routing"
   - Import `NextResponse` from "next/server"
   - Define route protection rules:
     * Public routes (no auth needed): /auth/login, /auth/register, /auth/forgot-password, /auth/reset-password, /auth/verify-email, /auth/oauth-role, /auth/403, / (home page)
     * Provider routes (require PROVIDER or ADMIN role): /provider/*
     * Admin routes (require ADMIN role): /admin/*
     * Authenticated routes (any logged-in user): /dashboard/*, /settings/*, /bookings/*
   - Middleware logic:
     a. First, run next-intl middleware for locale handling (get the intl response)
     b. Extract the pathname without locale prefix
     c. If the route is public, return the intl response as-is
     d. Get the JWT token via getToken({ req, secret: process.env.NEXTAUTH_SECRET })
     e. If no token (unauthenticated): redirect to /{locale}/auth/login?callbackUrl={originalUrl}
     f. If token exists but wrong role for the route: redirect to /{locale}/auth/403
     g. Otherwise: return the intl response (allow access)
   - Keep the existing config.matcher pattern that excludes api, _next/static, _next/image, favicon, etc.
   - IMPORTANT: The middleware must handle the locale prefix correctly. Pathnames from the request will include the locale (e.g., /fr/admin/dashboard). Strip the locale to check against route rules.

2. Add translation keys to fr.json under "auth" or "errors":
   - "accessDenied" ("Acces refuse")
   - "accessDeniedMessage" ("Vous n'avez pas les autorisations necessaires pour acceder a cette page.")
   - "backToDashboard" ("Retour au tableau de bord")
   - "backToLogin" ("Retour a la connexion")
  </action>

    Verify middleware correctly imports both next-intl and next-auth/jwt.
  </verify>
