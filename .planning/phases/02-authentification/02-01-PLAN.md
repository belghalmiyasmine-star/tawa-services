1. Install dependencies: `npm install next-auth@4 @auth/prisma-adapter bcryptjs` and `npm install -D @types/bcryptjs`. Use next-auth v4 (not v5/Auth.js) because v4 is stable with Next.js 15 App Router and has mature Prisma adapter support. The v5 beta has breaking API changes.

2. Update `src/env.ts` to add new env vars to the Zod schema:
   - GOOGLE_CLIENT_ID: z.string().optional() (optional so dev works without OAuth)
   - GOOGLE_CLIENT_SECRET: z.string().optional()
   - FACEBOOK_CLIENT_ID: z.string().optional()
   - FACEBOOK_CLIENT_SECRET: z.string().optional()
   - RESEND_API_KEY: z.string().optional() (needed for Plan 04)
   - NEXTAUTH_URL already exists in schema

3. Update `.env.example` with the new variables (no real values, just placeholders).

4. Create `src/types/next-auth.d.ts` -- TypeScript module augmentation:
   - Extend `next-auth` Session interface: add `user.id`, `user.role` (Role enum from @/types), `user.emailVerified` (boolean), `user.phoneVerified` (boolean)
   - Extend `next-auth` JWT interface: add same fields
   - This ensures useSession().data.user.role is typed

5. Create `src/lib/auth.ts` with NextAuth configuration:
   - Import PrismaAdapter from @auth/prisma-adapter, prisma from @/lib/prisma
   - Configure `authOptions: NextAuthOptions` with:
     a. adapter: PrismaAdapter(prisma)
     b. session: { strategy: "jwt", maxAge: 30 * 24 * 60 * 60 } (30 days per user decision)
     c. providers array:
        - CredentialsProvider with authorize callback that:
          * Accepts email + password
          * Finds user by email via prisma.user.findUnique
          * Compares password with bcryptjs.compare against user.passwordHash
          * Returns null if not found or password mismatch
          * Returns { id, email, name, role, emailVerified, phoneVerified } on success
        - GoogleProvider with clientId/clientSecret from env (conditionally added only if env vars exist)
        - FacebookProvider with clientId/clientSecret from env (conditionally added only if env vars exist)
     d. callbacks:
        - jwt callback: On sign-in, fetch user from DB and attach id, role, emailVerified, phoneVerified to token. On subsequent calls, pass through from token.
        - session callback: Copy id, role, emailVerified, phoneVerified from token to session.user
        - signIn callback: For OAuth providers, implement same-email auto-linking logic: if a user with the same email already exists, link the OAuth account to that user (create Account record) instead of creating a new user. If first-time OAuth and no existing user, allow sign-in (user creation handled by adapter). Set a flag in the token/URL to indicate "needs role selection" for first-time OAuth users (role defaults to CLIENT but can be changed).
     e. pages: { signIn: "/auth/login", error: "/auth/error" } (custom pages, locale prefix handled by middleware)
     f. secret: env.NEXTAUTH_SECRET
   - Export authOptions

6. Create `src/app/api/auth/[...nextauth]/route.ts`:
   ```ts
   import NextAuth from "next-auth";
   import { authOptions } from "@/lib/auth";
   const handler = NextAuth(authOptions);
   export { handler as GET, handler as POST };
   ```

7. Note: The CredentialsProvider authorize function is a placeholder that will be fully wired in Plan 02 (registration) and Plan 03 (login). For now it should correctly look up users and verify passwords -- it just won't have any users to find yet.
