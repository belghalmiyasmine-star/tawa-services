1. Update `src/lib/validations/auth.ts`:
   - Split `name` field into `firstName` (min 2, max 50) and `lastName` (min 2, max 50) -- per CONTEXT.md wizard step 2
   - Keep email, phone (tunisianPhoneSchema), password (8+ chars), confirmPassword, role fields
   - Add `acceptCGU: z.literal(true, { errorMap: () => ({ message: "Vous devez accepter les CGU" }) })` -- required at step 3
   - Keep the .refine for password match
   - Export updated RegisterFormData type

2. Create `src/features/auth/actions/register.ts`:
   - "use server" directive
   - Import prisma, registerSchema, bcryptjs
   - Create async function `registerAction(data: unknown): Promise<ActionResult<{ userId: string }>>`
   - Parse input with registerSchema.safeParse(data) -- return error if invalid
   - Check email uniqueness: prisma.user.findUnique({ where: { email } }) -- return French error "Cet email est deja utilise" if exists
   - Check phone uniqueness: prisma.user.findUnique({ where: { phone } }) -- return French error "Ce numero est deja utilise" if exists
   - Hash password with bcryptjs.hash(password, 12) (12 rounds as per bcrypt best practice)
   - Create user: prisma.user.create({ data: { email, passwordHash, phone, name: `${firstName} ${lastName}`, role } })
   - If role is PROVIDER, also create a Provider record: prisma.provider.create({ data: { userId, displayName: `${firstName} ${lastName}` } })
   - Return { success: true, data: { userId: user.id } }
   - Wrap in try/catch, return ActionResult error on failure

3. Add auth-related translation keys to `src/messages/fr.json` under the "auth" namespace:
   - Add keys for wizard steps: "registerTitle", "step1Title" ("Qui etes-vous?"), "step2Title" ("Vos informations"), "step3Title" ("Securisez votre compte"), "roleClient" ("Je cherche un service"), "roleProvider" ("Je propose mes services"), "firstName", "lastName", "acceptCGU", "cguLink", "registerButton", "alreadyHaveAccount", "loginLink"
   - Add keys for validation errors: "emailTaken", "phoneTaken", "invalidPhone", "passwordTooShort", "passwordMismatch", "cguRequired"
   - Preserve all existing keys -- only add new ones
