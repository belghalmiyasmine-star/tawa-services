  <action>
1. Add 2FA fields to the User model in `prisma/schema.prisma`:
   ```prisma
   twoFactorEnabled  Boolean   @default(false)
   twoFactorMethod   String?   // "TOTP" | "SMS"
   totpSecret        String?   // Encrypted TOTP secret
   ```
   Run `npx prisma generate`.

2. Install TOTP library: `npm install otpauth qrcode` and `npm install -D @types/qrcode`. Use the `otpauth` library for TOTP generation/verification (lightweight, no native dependencies, works in Edge runtime).

3. Create `src/features/auth/actions/setup-2fa.ts`:
   - "use server"
   - `setup2faAction(method: "TOTP" | "SMS")`:
     * For TOTP: generate a random secret using otpauth.Secret, create TOTP instance with issuer "Tawa Services" and account label = user email, generate otpauth:// URI, generate QR code as data URL using qrcode.toDataURL
     * Store the secret temporarily (e.g., in a session or a temporary DB record) until user confirms with a valid code
     * Return { qrCodeUrl, secret, otpauthUri } (secret shown as backup codes)
     * For SMS: just store the preference, OTP will be sent on each login via existing ISmsService
   - `confirm2faAction(code: string)`:
     * Verify the TOTP code against the stored secret
     * If valid: update user twoFactorEnabled = true, twoFactorMethod = "TOTP", totpSecret = secret
     * If invalid: return error

4. Create `src/features/auth/actions/verify-2fa.ts`:
   - "use server"
   - `verify2faLoginAction(userId: string, code: string, method: "TOTP" | "SMS")`:
     * For TOTP: verify code against user.totpSecret using otpauth
     * For SMS: verify against PhoneOtp record (reuse existing OTP verification logic)
     * Return ActionResult success/error

5. Create `src/features/auth/actions/disable-2fa.ts`:
   - "use server"
   - Requires current password verification before disabling
   - Update user: twoFactorEnabled = false, twoFactorMethod = null, totpSecret = null

6. Create `src/features/auth/components/TwoFactorSetup.tsx`:
   - "use client"
   - Two tabs: "Application d'authentification" (TOTP) and "SMS"
   - TOTP tab: shows QR code image, manual entry key, 6-digit verification input to confirm setup
   - SMS tab: explains that OTP will be sent to registered phone on each login, toggle to enable
   - "Desactiver la 2FA" section if already enabled (requires password confirmation)
   - All labels via useTranslations("auth")

7. Create `src/features/auth/components/TwoFactorChallenge.tsx`:
   - "use client"
   - Shown during login when user has 2FA enabled
   - Props: userId, method ("TOTP" | "SMS")
   - For TOTP: 6-digit code input, "Entrez le code de votre application d'authentification"
   - For SMS: 6-digit code input with "Code envoye au +216 ** *** X23", resend button with countdown
   - Submit calls verify2faLoginAction
   - On success: complete the sign-in (redirect to dashboard)
   - On failure: show error message

8. Create `src/app/[locale]/(client)/auth/2fa/page.tsx`:
   - Server component page for 2FA challenge during login
   - Receives userId and method from URL params or session state
   - Renders TwoFactorChallenge

9. Update `src/lib/auth.ts`:
   - In the signIn callback: after successful credential verification, check if user.twoFactorEnabled is true
   - If 2FA is enabled: instead of completing sign-in, redirect to /auth/2fa?userId=...&method=... (or use a temporary token in a cookie)
   - The actual JWT/session is only created after 2FA verification
   - For OAuth: same check -- after OAuth sign-in, if 2FA enabled, redirect to 2FA challenge

10. Add translation keys to fr.json: "twoFactorSetup", "twoFactorTotp", "twoFactorSms", "scanQrCode", "manualEntryKey", "confirmSetup", "twoFactorEnabled", "disableTwoFactor", "twoFactorChallenge", "enterAuthCode", "enterSmsCode"

  <action>
1. Add LoginRecord model to `prisma/schema.prisma`:
   ```prisma
   model LoginRecord {
     id        String   @id @default(cuid())
     userId    String
     ip        String?
     userAgent String?
     country   String?
     city      String?
     isNew     Boolean  @default(false)
     createdAt DateTime @default(now())

     @@index([userId])
     @@map("login_records")
   }
   ```
   Run `npx prisma generate`.

2. Create `src/lib/suspicious-login.ts`:
   - Export `checkSuspiciousLogin(userId: string, ip: string, userAgent: string): Promise<boolean>`:
     * Query the last 10 login records for this user
     * Check if the current IP or userAgent is new (not in previous records)
     * If new: return true (suspicious)
     * If not: return false
   - Export `recordLogin(userId: string, ip: string, userAgent: string): Promise<void>`:
     * Create LoginRecord with the current data
     * For IP geolocation: use a simple free service or skip geo for MVP -- just compare IP strings. Add comment "// TODO: Add IP geolocation via ipapi.co or similar for city/country"
   - Export `sendSuspiciousLoginEmail(userEmail: string, ip: string, userAgent: string, locale: string): Promise<void>`:
     * Use the existing email service (src/lib/email.ts) to send a notification
     * Email content: "Nouvelle connexion detectee depuis {ip} avec {userAgent}. Si ce n'etait pas vous, changez votre mot de passe immediatement."
     * Include a link to change password

3. Update `src/lib/auth.ts` signIn callback:
   - After successful authentication (credentials or OAuth), call recordLogin
   - Call checkSuspiciousLogin -- if suspicious, call sendSuspiciousLoginEmail
   - Extract IP from request headers (x-forwarded-for or connection.remoteAddress) and userAgent from request headers
   - Note: In Next.js middleware/API routes, access request headers via the request object. In the signIn callback, the request is available as the second argument.

4. Create `src/features/auth/components/SecuritySettings.tsx`:
   - "use client"

     a. "Authentification a deux facteurs" -- TwoFactorSetup component
     b. "Appareils connectes" -- list of recent login records (IP, device, date) from API endpoint
     c. "Changer le mot de passe" -- form with current password, new password, confirm new password
   - All via useTranslations("auth")

5. Create `src/app/[locale]/(client)/settings/security/page.tsx`:
   - Server component page
   - generateMetadata with "Securite" title
   - Fetch recent login records server-side
   - Render SecuritySettings with login records as prop
   - Protected by auth (requires logged-in user)

6. Add translation keys: "securityTitle", "connectedDevices", "changePassword", "currentPassword", "newDeviceAlert", "suspiciousLoginSubject", "suspiciousLoginBody", "passwordChanged"
