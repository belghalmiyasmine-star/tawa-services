  <action>
Import `sendNotification` from `@/features/notification/lib/send-notification` into each action file. Add fire-and-forget notification calls (void, never await to block the response) after each state change:

1. **manage-bookings.ts** — Add notifications after these actions:
   - `createBookingAction`: After booking created, notify provider: `sendNotification({ userId: providerUserId, type: "BOOKING_REQUEST", title: "Nouvelle demande de reservation", body: serviceTitle, data: { bookingId } })`.
   - `acceptBookingAction`: After booking accepted, notify client: `sendNotification({ userId: booking.clientId, type: "BOOKING_ACCEPTED", title: "Reservation acceptee", body: serviceTitle, data: { bookingId } })`.
   - `rejectBookingAction`: After booking rejected, notify client: `sendNotification({ userId: booking.clientId, type: "BOOKING_REJECTED", title: "Reservation refusee", body: serviceTitle, data: { bookingId } })`.
   - `completeBookingAction`: After booking completed, notify client: `sendNotification({ userId: booking.clientId, type: "BOOKING_COMPLETED", title: "Service termine", body: serviceTitle, data: { bookingId } })`.

2. **manage-quotes.ts** — Add notifications:
   - `createQuoteAction` (if exists, or wherever quote is created): Notify provider: `sendNotification({ userId: providerUserId, type: "QUOTE_RECEIVED", title: "Nouvelle demande de devis", body: serviceTitle, data: { quoteId } })`.
   - `respondQuoteAction`: Notify client: `sendNotification({ userId: quote.clientId, type: "QUOTE_RESPONDED", title: "Reponse a votre devis", body: serviceTitle, data: { quoteId } })`.
   - `acceptQuoteAction`: Notify provider: `sendNotification({ userId: providerUserId, type: "BOOKING_REQUEST", title: "Devis accepte, reservation creee", body: serviceTitle, data: { bookingId } })`.

3. **cancel-booking.ts** — Add notification:
   - After cancellation: Notify the OTHER party (if client cancels, notify provider; if provider cancels, notify client): `sendNotification({ userId: otherPartyId, type: "BOOKING_CANCELLED", title: "Reservation annulee", body: serviceTitle, data: { bookingId } })`.

4. **payment-actions.ts** — Find the payment release/complete action:
   - After payment released (RELEASED status): Notify provider: `sendNotification({ userId: providerUserId, type: "PAYMENT_RECEIVED", title: "Paiement recu", body: amount formatted, data: { bookingId, paymentId } })`.

5. **review-actions.ts** — Find the publish/submit review action:
   - After reviews are published (publishBothReviews or when both reviews exist): Notify both parties: `sendNotification({ userId: targetUserId, type: "REVIEW_RECEIVED", title: "Nouvel avis recu", body: stars + " etoiles", data: { bookingId, reviewId } })`.

6. **kyc-actions.ts** — Find approve/reject KYC actions:
   - After KYC approved: `sendNotification({ userId: provider.userId, type: "KYC_APPROVED", title: "Profil verifie", body: "Votre identite a ete verifiee", data: {} })`.
   - After KYC rejected: `sendNotification({ userId: provider.userId, type: "KYC_REJECTED", title: "Verification refusee", body: reason, data: {} })`.

7. **message-actions.ts** — In sendMessageAction, after message created:
   - Determine recipientId (the conversation participant who is NOT the sender).
   - `void sendNotification({ userId: recipientId, type: "NEW_MESSAGE", title: "Nouveau message", body: content.substring(0, 100), data: { conversationId, senderId: session.user.id } })`.

All notification calls must be fire-and-forget: `void sendNotification(...)` — use void prefix, never await in the main action flow. Wrap in try/catch inside sendNotification itself (already handled per Plan 02).

For each action, look up the service title, booking ID, and user IDs from the existing Prisma query result (already available in the action body — do NOT add extra DB queries).
