1. Create `src/features/messaging/actions/message-actions.ts`:
   - Add "use server" directive at top.
   - `sendMessageAction(formData: { conversationId: string; content: string }): Promise<ActionResult<{ messageId: string }>>`:
     - Get session via getServerSession(authOptions), return error if not authenticated.
     - Validate input with sendMessageSchema.
     - Fetch conversation with booking (include booking.status, booking.clientId, booking.service.providerId).
     - Verify current user is either the booking client or the provider's user — return error "unauthorized" if not.
     - Call `moderateMessageContent(content, booking.status)`. If blocked, return `{ success: false, error: "contact_info_blocked" }`.
     - Create message via `prisma.message.create({ data: { conversationId, senderId: session.user.id, content } })`.
     - Return `{ success: true, data: { messageId: message.id } }`.
   - `markMessagesAsReadAction(conversationId: string): Promise<ActionResult<{ count: number }>>`:
     - Get session, validate conversationId.
     - Verify user is a participant (same ownership check).
     - Update all unread messages where senderId !== session.user.id: `prisma.message.updateMany({ where: { conversationId, senderId: { not: session.user.id }, isRead: false }, data: { isRead: true, readAt: new Date() } })`.
     - Return count of updated messages.

2. Create `src/features/messaging/actions/conversation-queries.ts`:
   - Add "use server" directive at top.
   - `getConversationsAction(): Promise<ActionResult<ConversationListItem[]>>`:
     - Get session. Find all conversations where user is either booking.clientId or booking.service.provider.userId.
     - For each conversation, include: the other user's name and photo, the last message (content, createdAt, senderId), unread count (messages where senderId !== currentUser and isRead === false), booking info (service title, booking status).
     - Sort by last message createdAt DESC.
     - Return typed ConversationListItem[].
   - `getConversationMessagesAction(params: { conversationId: string; cursor?: string; limit?: number }): Promise<ActionResult<{ messages: MessageItem[]; nextCursor: string | null }>>`:
     - Validate with getConversationMessagesSchema.
     - Verify user is a participant.
     - Fetch messages with cursor-based pagination (orderBy: createdAt desc, take: limit + 1 for nextCursor detection).
     - Include sender name for each message.
     - Return messages sorted oldest-first (reverse after fetch) and nextCursor.
   - `getUnreadCountAction(): Promise<ActionResult<{ total: number }>>`:
     - Count all unread messages across all conversations where user is a participant and senderId !== currentUser.
   - `getOrCreateConversationAction(bookingId: string): Promise<ActionResult<{ conversationId: string }>>`:
     - Verify user is client or provider on the booking.
     - Use `prisma.conversation.upsert({ where: { bookingId }, create: { bookingId }, update: {} })`.
     - Return conversationId.
   - Define TypeScript interfaces in same file: `ConversationListItem { id, otherUser: { name, photoUrl }, lastMessage: { content, createdAt, isFromMe }, unreadCount, booking: { id, serviceTitle, status } }` and `MessageItem { id, senderId, senderName, content, createdAt, isRead, readAt, flagged }`.
