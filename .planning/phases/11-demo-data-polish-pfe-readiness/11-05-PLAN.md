**6. Create src/app/sitemap.ts:**
```ts
import type { MetadataRoute } from "next";

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const baseUrl = "https://tawa-services.tn";

  // Static pages
  const staticPages: MetadataRoute.Sitemap = [
    { url: baseUrl, lastModified: new Date(), changeFrequency: "daily", priority: 1.0 },
    { url: `${baseUrl}/fr/services`, lastModified: new Date(), changeFrequency: "daily", priority: 0.9 },
    { url: `${baseUrl}/fr/auth/login`, lastModified: new Date(), changeFrequency: "monthly", priority: 0.3 },
    { url: `${baseUrl}/fr/auth/register`, lastModified: new Date(), changeFrequency: "monthly", priority: 0.3 },
  ];

  // Dynamic category pages — fetch from DB
  // NOTE: For PFE demo this is sufficient. In production, use prisma to fetch all active categories.
  const categories = [
    "plomberie", "electricite", "menage-nettoyage", "cours-particuliers",
    "peinture-renovation", "demenagement", "jardinage", "climatisation",
    "serrurerie", "informatique-tech",
  ];
  const categoryPages: MetadataRoute.Sitemap = categories.map((slug) => ({
    url: `${baseUrl}/fr/services?category=${slug}`,
    lastModified: new Date(),
    changeFrequency: "weekly" as const,
    priority: 0.8,
  }));

  return [...staticPages, ...categoryPages];
}
```

NOTE: The sitemap uses hardcoded categories rather than Prisma queries to avoid importing the database client in the app router sitemap file (which runs at build time). For PFE this is sufficient.

Perform an accessibility audit and fix issues across key components:

**1. Navigation (Navbar + BottomNav + Footer):**
- Ensure `<nav>` wrapper with `aria-label="Navigation principale"` on Navbar
- Ensure `<nav>` wrapper with `aria-label="Navigation mobile"` on BottomNav
- Ensure `<footer>` with `role="contentinfo"`
- All icon-only buttons must have `aria-label` (e.g., theme toggle, notification bell, message icon)
- Verify DropdownMenu triggers have aria-haspopup and aria-expanded (radix handles this automatically — just verify)

**2. Search components:**
- SearchAutocomplete input: `aria-label="Rechercher un service"`, `role="combobox"`, `aria-expanded`, `aria-autocomplete="list"`
- Search results list: `role="listbox"`, each item `role="option"`
- SearchFilters labels: ensure every input has an associated label or aria-label

**3. Forms (Review, Booking):**
- StarRating: each star should have `aria-label="N etoile(s)"` and the group should have `role="radiogroup"` with `aria-label="Note"`
- ReviewForm: all fields should have visible labels or aria-labels
- AvailabilityCalendar: day buttons should have `aria-label` with full date (e.g., "Lundi 15 mars 2026")
- All form error messages should use `aria-describedby` linking to the input

**4. Images:**
- Audit all `<img>` and `<Image>` tags for alt text
- Service photos: alt="Photo du service {title}"
- Provider avatars: alt="Photo de {displayName}"
- Decorative images (icons, backgrounds): alt="" (empty string, not missing)
- Logo: alt="Tawa Services"

**5. Semantic HTML:**
- Ensure pages use proper heading hierarchy (h1 -> h2 -> h3, no skipped levels)
- Homepage should have exactly one h1 ("Trouvez un prestataire..." or similar)
- Each page should have a `<main>` element wrapping the primary content
- Use `<section>` with aria-label for distinct page sections

**6. Color contrast:**
- Verify text on colored backgrounds meets WCAG AA (4.5:1 for normal text, 3:1 for large text)
- Check badge text contrast (e.g., status badges, trust badges)
- If any contrast issues found, adjust the Tailwind color class (e.g., text-gray-600 -> text-gray-700)

**7. Focus management:**
- Ensure all interactive elements are keyboard-accessible (Tab key navigation)
- Ensure focus-visible ring is visible on all focusable elements (Tailwind's focus-visible:ring-2 pattern)
- Modal/Dialog components should trap focus (Radix handles this — verify)
- After dialog closes, focus should return to trigger element (Radix handles this — verify)

Do NOT over-engineer — fix only clear accessibility violations. The goal is Lighthouse Accessibility 75+, not WCAG AAA compliance.
