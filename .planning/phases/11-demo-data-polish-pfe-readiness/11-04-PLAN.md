**1. Update next.config.ts:**
Add image optimization configuration:
```ts
images: {
  formats: ['image/avif', 'image/webp'],
  deviceSizes: [375, 640, 768, 1024, 1280],
  imageSizes: [16, 32, 48, 64, 96, 128, 256],
  // Allow local uploads directory
  remotePatterns: [
    {
      protocol: 'https',
      hostname: '**',
    },
  ],
},
```

Add experimental optimizations if not already present:
```ts
experimental: {
  optimizePackageImports: ['lucide-react', 'recharts'],
},
```

The `optimizePackageImports` ensures tree-shaking works for lucide-react (only imports used icons) and recharts (large library).

**2. Convert img tags to Next.js Image component where feasible:**
Search for `<img` tags in key components. For service photos, provider avatars, and review photos:
- Replace `<img src={url} ...>` with `<Image src={url} alt={...} width={N} height={N} loading="lazy" className={...} />`
- Import from `next/image`
- Set appropriate width/height for the layout context
- Use `loading="lazy"` for below-fold images (default in Next.js Image)
- Use `priority` prop for above-fold hero images (first service card image, provider profile photo)

**Important caveats:**
- Many images use `/public/uploads/...` paths — these work with Next.js Image since they're local
- For empty photoUrls (seed data has none), the Image component should not render — keep the existing placeholder/fallback
- If a component uses photoUrl that could be null, keep the existing conditional rendering
- Do NOT convert images inside components that use complex CSS (background-image, etc.) — only convert standard img tags

**3. Add loading="lazy" to any remaining native img tags** that can't be easily converted to next/image (e.g., inside rich text, or dynamic content).

**4. Verify no unnecessarily large bundle imports:**
Check that recharts is only imported in admin analytics pages (not in the main client bundle). If any recharts component is imported in a shared layout, wrap it in dynamic import:
```tsx
const Chart = dynamic(() => import('./Chart'), { ssr: false });
```
